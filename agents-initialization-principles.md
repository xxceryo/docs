# AGENTS.md 初始化原则

## 目标

`AGENTS.md` 用于帮助 Agent 快速理解项目边界、执行方式和约束。初始化时应建立一套层次清晰、可执行、可验证且易于维护的规则体系，而不是将所有项目说明集中堆叠在单个文件中。

## 核心原则

### 1. 根文件只作为规则入口

根目录 `AGENTS.md` 应保持简洁，仅包含：

- 规则适用范围。
- 主题规则文件索引。
- 不同任务需要读取哪些规则。
- 上级规则、嵌套规则和用户指令的优先关系。

具体开发规范应按主题放入独立规则文件，避免根文件过长后难以定位和维护。

### 2. 按明确职责拆分规则

每个规则文件只负责一个明确主题。常见主题包括：

- 项目结构、技术栈和目录职责。
- 开发环境、依赖管理和常用命令。
- 测试策略、验证命令和完成标准。
- 各技术栈的编码规范。
- Git 安全、分支和提交规范。
- Agent 规则自身的维护规范。

不同主题之间应通过索引或链接建立关系，不应重复维护同一条规则。

### 3. 规则必须匹配项目现状

初始化规则前，应先检查仓库当前的目录结构、依赖声明、构建工具、测试工具和常用脚本。规则只能描述项目实际采用的技术与流程，不应包含失效说明或尚未采用的假设方案。

项目结构、技术栈或开发流程变化后，应同步更新相关规则。

### 4. 使用可执行、可验证的表述

规则应明确说明 Agent 应做什么、何时执行以及如何验证。例如：

- “后端改动完成前运行 `uv run pytest`。”
- “新增后端依赖使用 `uv add <package>`，并同步提交锁文件。”
- “修改代码前读取相关模块和测试。”

避免使用“适当处理”“注意质量”“视情况测试”等无法判断是否完成的模糊表述。

### 5. 明确规则的适用范围和优先级

根规则默认适用于整个仓库。仅当某个子目录确实存在不同技术栈、命令或约束时，才在该目录创建嵌套 `AGENTS.md`。

嵌套规则应：

- 明确自身适用的目录范围。
- 只补充或覆盖必要规则。
- 明确说明有意覆盖的上级规则。
- 避免重复上级规则中仍然有效的内容。

直接用户指令优先于仓库规则；更深层目录的规则可以补充或覆盖上级规则。

### 6. 保持规则安全且可共享

规则文件不得包含：

- 密钥、口令、令牌或其他敏感信息。
- 开发者个人路径或仅适用于个人机器的配置。
- 个人偏好。
- 仅适用于单次任务的临时要求。

本地环境示例应使用占位值，并将真实敏感配置保留在不提交的环境文件中。

### 7. 控制规则数量和复杂度

只记录能够持续帮助 Agent 正确执行任务的规则。对代码已经清晰表达、工具默认能够处理或无法验证的内容，不必额外制定规则。

新增规则前应确认：

- 该规则解决了真实且可重复出现的问题。
- 该规则应由 Agent 遵守，而不是仅供人类阅读。
- 该规则没有在其他文件中重复定义。
- 该规则能够被实际执行或验证。

## 推荐目录结构

```text
AGENTS.md
agents/
├── project.md
├── development.md
├── testing.md
├── coding-style.md
├── frontend-coding-style.md
├── git.md
└── agents-maintenance.md
```

项目可根据实际技术栈增删主题文件，但应保持根文件作为入口、主题文件各负其责的结构。

## 初始化步骤

1. 检查项目目录、依赖文件、构建脚本、测试配置和现有文档。
2. 识别项目的技术栈、目录职责、开发命令和验证方式。
3. 创建根目录 `AGENTS.md`，定义适用范围、规则索引和读取顺序。
4. 按主题创建规则文件，记录项目实际采用的约束。
5. 为修改代码、测试、Git 操作等常见任务定义必须读取的规则。
6. 检查是否存在需要独立规则的子项目或特殊目录。
7. 验证所有链接、命令、目录名称和规则描述。

## 根 AGENTS.md 最小内容

一个可用的根 `AGENTS.md` 至少应包含：

```markdown
# Agent Rules Index

本文件适用于整个仓库，仅作为 Agent 规则入口。

## Rule Files

- [项目结构与职责](agents/project.md)
- [开发规范](agents/development.md)
- [测试规范](agents/testing.md)
- [编码规范](agents/coding-style.md)
- [Git 规范](agents/git.md)
- [Agent 规则维护规范](agents/agents-maintenance.md)

## Usage

1. 始终读取本索引。
2. 根据当前任务读取相关规则。
3. 修改代码时读取项目、开发、测试和对应技术栈的编码规范。
4. 涉及 Git 操作时读取 Git 规范。
5. 更深层目录中的 `AGENTS.md` 可补充或覆盖规则；直接用户指令优先级最高。
```

该模板仅定义结构。初始化时必须根据项目的实际目录、技术栈和工作流程调整内容。

## 验证清单

- 根 `AGENTS.md` 只包含入口、索引、适用范围和读取顺序。
- 所有索引链接均指向存在的文件。
- 每个规则文件只负责一个明确主题。
- 规则与当前目录结构、技术栈和工具一致。
- 常用命令能够在项目环境中执行。
- 规则描述明确、可执行且可验证。
- 不同文件之间不存在重复或冲突规则。
- 嵌套规则的适用范围和覆盖关系清晰。
- 文件中不包含敏感信息、个人配置或临时要求。
- `git diff` 中不包含与规则初始化无关的修改。
